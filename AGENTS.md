## CONVENTIONS

### Conditional Compilation
- `#ifdef WL_CUDA` ‚Äî gates ALL CUDA code. Defined in xmake.lua when `CUDA_PATH` is set.
- `#ifdef WL_OPTIX` ‚Äî gates OptiX denoiser code. Defined in xmake.lua when OptiX SDK detected.
- `#ifdef WL_ISPC` ‚Äî gates ISPC SIMD code. Defined in xmake.lua when `vendor/ispc/bin/ispc.exe` exists.
- `#ifdef WL_DEBUG` / `WL_RELEASE` / `WL_DIST` ‚Äî per-configuration defines.
- `#ifndef WL_CUDA` ‚Äî CPU-only path (`Renderer.cpp`). ISPC path is nested inside this block when `WL_ISPC` is also defined.

### Memory Layout Contracts (CRITICAL)
- `GPUPackedMaterial` (host, `CUDARenderer.h:108`) ‚Üî `GPUMaterial` (device, `CUDATypes.cuh:27`) ‚Äî 36 bytes, identical layout
- `GPUPackedSphere` (host, `CUDARenderer.h:97`) ‚Üî `GPUSphere` (device, `CUDATypes.cuh:16`) ‚Äî 20 bytes, identical layout
- `float3` (host, `CUDARenderer.h:89`) ‚Üî CUDA `float3` ‚Äî both 12 bytes, 4-byte alignment
- **ANY layout change must update BOTH sides simultaneously.**

### Scene Data Flow
- ImGui modifies `Scene` directly via structured bindings (`glm::value_ptr`)
- CPU path reads `m_ActiveScene` pointer ‚Üí always gets latest data
- GPU path: `UploadSceneToGPU()` called when `scene.Version` changes ‚Üí cudaMemcpy host‚Üídevice
- GPU realloc only when sphere/material counts change (`CUDARenderer.cu:158-219`)
- Scene version tracking: `Scene::Version` incremented on any ImGui change; `m_LastSceneVersion` compared to skip uploads

### Rendering Pipeline (GPU)
1. Scene data upload (if version changed) ‚Üí sphere/materials H2D
2. Camera position + ray directions upload (if dirty)
3. `RenderKernel` ‚Üí raw path trace samples ‚Üí `d_SampleBuffer`
4. `PostProcessKernel` ‚Üí accumulate + average + clamp + RGBA ‚Üí output buffer
5. (Optional) OptiX denoiser on `d_DenoiseBuffer` ‚Üí `ConvertToRGBAKernel`
6. Output: interop path ‚Üí `vkCmdCopyBufferToImage`; legacy path ‚Üí D2H + `SetData`

### Emission Integration Order
- Both CPU and GPU: `light += contribution * emission` BEFORE BRDF updates contribution
- Matches path tracing integral: Le ¬∑ ‚àè(previous BSDFs)
- With GGX BRDF: contribution updated via `(spec + diff) * NdotL / pdf` (replaces old `contribution *= albedo`)

### Naming
- PascalCase for classes/structs, camelCase for methods/members
- `m_` prefix for member variables (`m_Camera`, `m_FrameIndex`)
- GPU types prefixed with `GPU` (`GPUSphere`, `GPUMaterial`, `GPUScene`, `GPUHitPayload`)
- Host-side GPU packing structs prefixed with `GPUPacked`
- CUDA interop: C API functions prefixed with `CUDARenderer_`; RAII class `VkCUDAInterop`

## ANTI-PATTERNS

- **NEVER modify Walnut library files** ‚Äî extend via `ExampleLayer` subclass (exceptions: Vulkan extensions require minimal changes to `Application.cpp`/`Image.h`, see CONVENTIONS)
- **NEVER suppress CUDA errors** ‚Äî always check `cudaMalloc`/`cudaMemcpy`/`cudaStreamCreate` return values
- **NEVER assume scene data is synced to GPU** ‚Äî increment `Scene::Version` on any scene change
- **NEVER change `GPUPacked*` struct layout without updating `GPUMaterial`/`GPUSphere` in `CUDATypes.cuh`**
- **NEVER use `--remote` on `git submodule update`** ‚Äî vendor deps are pinned by Walnut
- **NEVER create/destroy VkCUDAInterop during CUDA kernel execution** ‚Äî set up before render, destroy after sync
- **NEVER call OptiXDenoiser::Denoise before Initialize** ‚Äî check `IsValid()` first
- **NEVER assume non-interop output path is active** ‚Äî check `m_InteropEnabled` before using `CUDARenderer_GetOutput`

## UNIQUE STYLES

- Structured bindings for sphere/material field access: `auto& [Position, Radius, MaterialIndex] = spheres[i]`
- Box-drawing comment separators: `// ‚îÄ‚îÄ‚îÄ Section Title ‚îÄ‚îÄ‚îÄ`
- No sky/environment light ‚Äî all illumination from emissive materials only
- `m_NeedsRender` flag for conditional re-render (ImGui change detection)
- `m_RayDirsDirty` flag for lazy camera ray direction re-upload
- `Scene::Version` for dirty-tracked GPU scene upload dedup
- Two-stage GPU pipeline: raw sample ‚Üí accumulation + post-process (separate kernels on same stream)
- Dual output path: interop (zero-copy Vulkan buffer) vs legacy D2H copy
- OptiX denoiser runs after accumulation, before RGBA conversion

## NOTES

- **No tests exist** ‚Äî verification is manual (visual inspection of rendered output)
- **Single light source** ‚Äî only emissive materials illuminate the scene; no env/sky light
- **GPU accumulation** ‚Äî old samples persist in buffer; use Reset button or disable Accumulate for clean re-render after ImGui changes
- **CUDA arch fallback** ‚Äî if nvcc fails, check GPU compute capability matches `sm_XX` in `xmake.lua:130-133`
- **ISPC arch fallback** ‚Äî if ISPC not found, CPU path falls back to C++ `std::execution::par` (no SIMD)
- **Walnut submodule** ‚Äî upstream fork: `https://github.com/Cle2ment/Walnut`; vendor deps pinned to tested commits
- **ISPC download** ‚Äî `scripts\Setup.bat` auto-downloads to `vendor\ispc\`; manual: https://github.com/ispc/ispc/releases/tag/v1.30.0
- **OptiX denoiser** ‚Äî requires OptiX SDK 7.0+; detected via `OptiX_ROOT` or `OPTIX_PATH` env var, falling back to `C:\ProgramData\NVIDIA Corporation\OptiX SDK*\`
- **Vulkan-CUDA interop** ‚Äî Windows-only (Win32 external memory handles); toggle via ImGui checkbox; falls back to D2H path if interop creation fails
- **GGX Microfacet BRDF** ‚Äî replaces Lambertian diffuse on all three paths (CUDA, CPU C++, ISPC SMID); uses roughness¬≤ remapping; metallic workflow with Fresnel-Schlick
